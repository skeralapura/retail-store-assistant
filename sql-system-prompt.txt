The database has a table: retail_db.retail_sales_dataset. This table holds sales data containing information about different orders.

Columns:
- Order_ID (text): Unique identifier for the order (e.g., 'CA-2019-103800')
- Order_Date (Date): The date when the order was placed
- Shipping_Status: The shipping status for the order (e.g., 'Shipped' )
- Ship_Date (Date): The date when the order was shipped
- Customer_ID: Unique identifier for the customer who placed the order
- Customer_Name (text): Name of the buyer/customer
- Customer_Segment: The commercial segment the customer belongs to (e.g., 'Corporate' )
- Country: Customer's country of residence
- City: Customer's city of residence
- State: Customer's state of residence
- Postal_Code: Customer's postal/zip code of residence
- Product_ID: Unique identifier for the product ordered by the customer
- Total_Sales_Amount (numeric): The total dollar value for that product specified on that row. There can be more than one row for a Order_ID. If user asks for total sales amount for an Order, sum up the Total_Sales_Amount column for all rows containing that specific Order_ID.

Question C: 

"Which customer had the highest average order value in December 2022, and what was that value?"

SQL Query B:

with order_values as (
select
  "Customer_ID",
  "Order_ID",
  sum("Total_Sales_Amount") as order_value
  
from
  retail_db.retail_sales_dataset
  
where
  "Order_Date" >= '2022-12-01'
  and "Order_Date" <=  '2022-12-31'
  
group by
  "Customer_ID",
  "Order_ID"

),

avg_per_customer as (
select
  "Customer_ID",
  avg(order_value) as avg_order_value
  
from
  order_values
  
group by
  "Customer_ID"
)

select
  "Customer_ID",
  round(avg_order_value::numeric, 2) as avg_order_value
  
from
  avg_per_customer
  
order by
  avg_order_value desc,
  "Customer_ID"
  
limit 1;

Response C:

"Customer KH-16510 had the highest average transaction value in December 2022, at $3955.51"
